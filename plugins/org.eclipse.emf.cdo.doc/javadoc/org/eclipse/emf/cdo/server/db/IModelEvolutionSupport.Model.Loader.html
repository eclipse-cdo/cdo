Schnittstelle IModelEvolutionSupport.Model.Loader

IModelEvolutionSupport.Model
public static interface IModelEvolutionSupport.Model.Loader
Loads models stored in a DB store.

Model loaders are registered via factories contributing to the org.eclipse.emf.cdo.server.db.modelLoaders product group. Here is an example contribution in plugin.xml: 

 <extension point="org.eclipse.net4j.util.factories">
   <annotationFactory
     productGroup="org.eclipse.emf.cdo.server.db.modelLoaders"
     type="default"
     productClass="org.eclipse.emf.cdo.server.internal.db.DefaultModelLoader"/>
 </extension>
Model loaders are used by model evolution support implementations to load the models stored in a DB store and to provide the corresponding registered EPackages (if available).

Note that model evolution happens very late during the activation of a DB store, but very early during the activation of the IRepository. Therefore, model loaders must not rely on any services of the repository. In particular, they must not access the repository's package registry because it is not yet available!
